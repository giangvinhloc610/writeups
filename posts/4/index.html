The MRE platform

Part 1: A bit background of the MRE platform


1. What is MRE platform?

This is how MRE defined itself:

MRE (MAUI Runtime Environment) is a phone application development platform similar to JVM and Brew. On the MRE platform, you can realize solutions for smart featuer phones on feature phones. Meanwhile, MediaTek also provides devlopers and end suppliers with highly efficient development tools (MRE SDK) and compilation environment for applications, allowing developers to develop applications more quickly and effectively.

Source here (we will talk about this later)

TL;DR: It's an OS/Runtime environment developed by Mediatek for cellphones based on MTK chips.

2. What is it used for?

If you have phones in the Nokia S30+ series, like Nokia 220, Nokia 225, ... (see full list here), then you have a phone running MRE platform!

Some other MRE phones include cheap some cheap Chinese phones (I'm not sure of their names), some phone for older people (which has large screen and big text, along with read-aloud features). I have seen old Viettel phone running MRE, too :))

If you open up some handheld phone (not Android phone) and see Mediatek chip (especially MT62xx series), then it might run MRE platform

Some development board (like the Linkit One) also based on MRE platform.

3. Current status of MRE

MRE has been abandoned by Mediatek since 2016. People have claimed here that it's a market failure. The website, including the SDK, documentation, forum, disscussion and other things were totally deleted from the Internet since then (Wayback Machine archived some of the pages but not all).

4. MRE "OS", MRE runtime and MRE SDK

MRE can be called an OS, which was based on Nucleus RTOS.

MRE runtime is the runtime of MRE, for running .vxp apps on the OS, and provides a set of APIs for the app. It supports application written in C, and C++, or anything that can be compiled to ARM machine code (we will talk about this further in the next part of the series).

Like other OSes, MRE has it own SDK, supports C and C++. This SDK has been removed from the offical page, but luckily some man still keep it on Github (talk about this later)

5. MRE as a replacement for J2ME

Actually, MRE is developed to replace the J2ME platform, and it was choosen by Microsoft after taking over Nokia to replace old Symbian OS.
